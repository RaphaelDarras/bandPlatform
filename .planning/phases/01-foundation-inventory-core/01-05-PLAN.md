<objective>
Fix PUT /api/products/:id to safely update product details and variant metadata without corrupting stock or version data managed by /api/inventory endpoints.

Purpose: The current PUT endpoint passes the request body (minus only top-level `stock`) directly to `findByIdAndUpdate`. If the body contains a `variants` array, MongoDB replaces all embedded variant documents, destroying `stock` and `version` fields. This makes it impossible for admins to update product names, prices, or variant colors/sizes without risking inventory data corruption.

Output: A rewritten PUT handler in `api/routes/products.js` that whitelists safe fields at both the product level and the variant level.
</objective>

Replace the PUT /:id handler (lines 147-173) in api/routes/products.js with a safe implementation:

1. Define allowed product-level fields: `['name', 'description', 'basePrice', 'category', 'images', 'active']`
2. Define allowed variant metadata fields: `['size', 'color', 'priceAdjustment']`
3. Build a `$set` object from the request body:
   - For each allowed product-level field present in req.body, add to $set (e.g., `{ name: "New Name" }`)
   - If `variants` array is present in req.body, do NOT pass it to $set. Instead, handle variant updates separately.
4. For variant metadata updates:
   - First, apply the product-level $set using `findByIdAndUpdate` if there are product-level fields
   - Then, for each variant in req.body.variants that has a `sku` field, use a targeted update to set ONLY the allowed metadata fields on the matching embedded variant:
     ```javascript
     // For each variant in the request:
     const variantUpdate = {};
     for (const field of allowedVariantFields) {
       if (variant[field] !== undefined) {
         variantUpdate[`variants.$.${field}`] = variant[field];
       }
     }
     if (Object.keys(variantUpdate).length > 0) {
       await Product.updateOne(
         { _id: req.params.id, 'variants.sku': variant.sku },
         { $set: variantUpdate }
       );
     }
     ```
   - This uses positional operator `$` to update only the matched variant, and only the whitelisted metadata fields.
5. After all updates, fetch and return the updated product with `Product.findById(req.params.id)`.
6. If no product found by id, return 404.
7. If req.body is empty or has no recognized fields, return 400 with message "No valid update fields provided".
8. Update the Swagger JSDoc for PUT /:id to document:
   - That stock and version fields are protected and cannot be modified through this endpoint
   - That variant updates match by SKU and only update size, color, priceAdjustment
   - 400 response for empty/invalid body

Important: Do NOT use `findByIdAndUpdate(id, req.body)` or any pattern that passes unfiltered body content. Every field written to the database must be explicitly whitelisted.
