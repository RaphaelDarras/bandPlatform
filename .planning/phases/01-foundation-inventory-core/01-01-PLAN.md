<task type="auto">
  <name>Task 3: Create Mongoose schemas with optimistic locking support</name>
  <files>
    api/models/Product.js

    api/models/Concert.js
  </files>
  <action>
Create three Mongoose schemas following user's MongoDB design decisions from 01-CONTEXT.md:

**Product.js (per-variant optimistic locking):**
- Schema fields: name (String, required), description (String), category (String), basePrice (Number, required), images ([String]), active (Boolean, default: true), createdAt/updatedAt (timestamps)
- Embedded variants array with schema: sku (String, required, unique within product), size (String), color (String), stock (Number, default: 0), version (Number, default: 0), priceAdjustment (Number, default: 0)
- Index on "variants.sku" for fast variant lookups
- Index on "variants.version" to support optimistic locking queries
- Export model: `module.exports = mongoose.model('Product', ProductSchema)`

**Order.js (with inventory audit trail):**
- Schema fields: orderNumber (String, unique), customerEmail (String, required), customerName (String), items array, totalAmount (Number, required), status (enum: ['pending', 'paid', 'failed', 'shipped', 'delivered'], default: 'pending'), paymentMethod (enum: ['stripe', 'paypal']), paymentIntentId (String), source (String, default: 'online'), createdAt/updatedAt (timestamps)
- Items array schema: productId (ObjectId, ref: 'Product'), variantSku (String, required), quantity (Number, required, min: 1), priceAtPurchase (Number, required), stockBefore (Number, required), stockAfter (Number, required)
- The stockBefore/stockAfter fields per item provide inventory audit trail (per user decision: no separate Inventory collection)
- Index on orderNumber for lookups
- Export model: `module.exports = mongoose.model('Order', OrderSchema)`

**Concert.js (for sales attribution):**
- Schema fields: name (String, required), date (Date, required), location (String, required), venue (String), active (Boolean, default: true), createdAt/updatedAt (timestamps)
- Index on date (descending) for recent concerts first
- Export model: `module.exports = mongoose.model('Concert', ConcertSchema)`

**Note on schema decisions:**
- Using embedded variants per user decision (single Products collection, not separate Variants)
- Per-variant version field enables concurrent updates to different sizes without conflict
- Embedded stockBefore/stockAfter in Orders provides audit trail per user decision (no separate Inventory collection)
- Sales model will be created in Plan 03 alongside API endpoints
  </action>
