**Authentication routes (api/routes/auth.js):**
- Import: express.Router, jwt, Admin model
- POST /login endpoint:
  - Accept JSON body: { username, password }
  - Validate required fields, return 400 if missing
  - Find admin: `Admin.findOne({ username, active: true })`
  - If not found, return 401: `{ error: 'Invalid credentials' }`
  - Compare password using admin.comparePassword(password)
  - If password invalid, return 401: `{ error: 'Invalid credentials' }`
  - Update lastLogin: `admin.lastLogin = new Date()`
  - Generate JWT: `jwt.sign({ userId: admin._id, username: admin.username, role: admin.role }, process.env.JWT_SECRET, { expiresIn: '24h' })`
  - Return 200: `{ token, user: { id: admin._id, username: admin.username, role: admin.role } }`
- POST /verify endpoint (optional, for token validation):
  - Use authenticateToken middleware
  - Return 200: `{ valid: true, user: req.user }`
- Export router

**Product routes (api/routes/products.js):**
- Import: express.Router, Product model, authenticateToken middleware
- GET / (public - no auth required):
  - Query all active products: `Product.find({ active: true })`
  - Optionally filter by category: `?category=apparel`
  - Return 200: array of products with all variants
- GET /:id (public):
  - Find product by ID: `Product.findById(req.params.id)`
  - Return 404 if not found
  - Return 200: product object with variants
- POST / (protected - requires authenticateToken):
  - Create new product from request body
  - Validate required fields: name, basePrice, variants array
  - Each variant must have sku, size or color
  - Return 201: created product
- PUT /:id (protected):
  - Update product by ID: `Product.findByIdAndUpdate()`
  - Allow updating name, description, basePrice, images, active status
  - Do NOT allow direct stock updates (use inventory route for stock changes)
  - Return 200: updated product
- DELETE /:id (protected):
  - Soft delete: set active = false instead of removing document
  - Return 200: `{ message: 'Product deactivated' }`
- Export router

**Error handling:**
Wrap async handlers in try-catch, return 500 with error message on unexpected failures.

<task type="auto">
  <name>Task 3: Create inventory management routes with optimistic locking and explicit audit trail</name>
  <files>
    api/routes/inventory.js
  </files>
  <action>
Create inventory routes implementing atomic stock operations using optimistic locking with EXPLICIT audit trail creation:

**Setup:**
- Import: express.Router, Product, Order, Sale, Concert, authenticateToken
- All routes require authenticateToken (admin only)

**POST /deduct endpoint:**
Primary stock deduction endpoint used by both online shop and POS:
- Request body: `{ productId, variantSku, quantity, source: 'online' | 'pos', metadata: { orderId?, saleId?, concertId?, customerEmail?, totalAmount, priceAtPurchase } }`
- Find product and specific variant: `Product.findOne({ _id: productId, 'variants.sku': variantSku })`
- Extract matching variant from variants array: `const variant = product.variants.find(v => v.sku === variantSku)`
- Capture stockBefore: `const stockBefore = variant.stock`
- Check sufficient stock: `if (stockBefore < quantity) return res.status(409).json({ error: 'Insufficient stock', available: stockBefore })`
- **Optimistic locking update:**
  - Use `Product.findOneAndUpdate()` with atomic operators
  - Match conditions: `{ _id: productId, 'variants.sku': variantSku, 'variants.$.version': variant.version }`
  - Update operations: `{ $inc: { 'variants.$.stock': -quantity, 'variants.$.version': 1 } }`
  - Options: `{ new: true }` to return updated document
  - If result is null, version conflict detected → return 409: `{ error: 'Stock conflict - version mismatch. Please retry with updated stock data.' }`
- Capture stockAfter from updated variant: `const updatedVariant = updatedProduct.variants.find(v => v.sku === variantSku); const stockAfter = updatedVariant.stock`
- **EXPLICIT audit trail creation (CRITICAL):**
  - If source === 'online':
    ```javascript
    await Order.create({
      orderNumber: metadata.orderId || `ORD-${Date.now()}`,
      customerEmail: metadata.customerEmail,
      items: [{
        productId,
        variantSku,
        quantity,
        priceAtPurchase: metadata.priceAtPurchase,
        stockBefore,
        stockAfter
      }],
      totalAmount: metadata.totalAmount,
      status: 'pending',
      source: 'online'
    });
    ```
  - If source === 'pos':
    ```javascript
    await Sale.create({
      concertId: metadata.concertId,
      items: [{
        productId,
        variantSku,
        quantity,
        priceAtSale: metadata.priceAtPurchase,
        stockBefore,
        stockAfter
      }],
      totalAmount: metadata.totalAmount,
      paymentMethod: metadata.paymentMethod || 'cash',
      source: 'pos',
      createdBy: req.user.userId
    });
    ```
- Return 200: `{ success: true, stockAfter, version: updatedVariant.version, auditId: createdOrder._id or createdSale._id }`

**POST /reserve endpoint:**
Reserve stock for online checkout (10-minute timeout per 01-CONTEXT.md):
- Request body: `{ productId, variantSku, quantity, reservationId }`
- Find product and variant
- Check sufficient stock
- **Optimistic locking pattern (same as /deduct):**
  - Match with version: `{ _id: productId, 'variants.sku': variantSku, 'variants.$.version': currentVersion }`
  - Decrement stock: `{ $inc: { 'variants.$.stock': -quantity, 'variants.$.version': 1 } }`
- **Important: Reserve/release do NOT create audit entries** - These are temporary holds. Only final /deduct after payment creates Order document with audit trail.
- Store reservation metadata in-memory or simple collection with TTL index (expires after 10 minutes)
- Return 200: `{ reserved: true, expiresAt: Date.now() + 600000, version: updatedVariant.version }`

**POST /release endpoint:**
Release reserved stock if payment fails or times out:
- Request body: `{ productId, variantSku, quantity, reservationId, currentVersion }`
- **Optimistic locking pattern:**
  - Match with version: `{ _id: productId, 'variants.sku': variantSku, 'variants.$.version': currentVersion }`
  - Increment stock back: `{ $inc: { 'variants.$.stock': +quantity, 'variants.$.version': 1 } }`
- **Important: No audit entry created** - Reservation releases don't count as sales, so no Order/Sale document.
- Remove reservation from store
- Return 200: `{ released: true, stockAfter: updatedVariant.stock, version: updatedVariant.version }`

**GET /audit endpoint:**
Query inventory audit trail from Order/Sale documents:
- Query params: `?productId={id}&startDate={date}&endDate={date}`
- Aggregate from Order.items and Sale.items where productId matches:
  ```javascript
  const orderAudits = await Order.aggregate([
    { $unwind: '$items' },
    { $match: { 'items.productId': productId, createdAt: { $gte: startDate, $lte: endDate } } },
    { $project: { timestamp: '$createdAt', source: 'online', quantity: '$items.quantity', stockBefore: '$items.stockBefore', stockAfter: '$items.stockAfter', orderId: '$_id' } }
  ]);
  const saleAudits = await Sale.aggregate([
    { $unwind: '$items' },
    { $match: { 'items.productId': productId, createdAt: { $gte: startDate, $lte: endDate } } },
    { $project: { timestamp: '$createdAt', source: 'pos', quantity: '$items.quantity', stockBefore: '$items.stockBefore', stockAfter: '$items.stockAfter', saleId: '$_id' } }
  ]);
  ```
- Combine and sort by timestamp
- Return 200: audit log array `[{ timestamp, source, quantity, stockBefore, stockAfter, orderId or saleId }]`

**Error handling:**
- All routes wrapped in try-catch
- Database errors → 500
- Validation errors → 400
- Stock conflicts → 409

**Critical implementation notes:**
- The `variants.$` positional operator updates matched array element atomically
- Version check in match condition ensures update fails if another transaction modified variant
- Per user decision from 01-CONTEXT.md: fail immediately on conflict, no retry logic
- This pattern prevents last-item overselling race condition (satisfies INV-02)
- Reserve/release are temporary holds - they modify stock but DON'T create audit entries
- Only final deduction (POST /deduct) creates Order/Sale documents with stockBefore/stockAfter audit trail

Export router.
