// 2. Atomic update with version check
const updatedProduct = await Product.findOneAndUpdate(
  { _id: productId, 'variants.sku': variantSku, 'variants.$.version': variant.version },
  { $inc: { 'variants.$.stock': quantity, 'variants.$.version': 1 } },

3. Add POST /restock route in `api/routes/inventory.js` (place it after the /release route and before the GET /audit route). Follow the EXACT same optimistic locking pattern as /deduct but with positive $inc for stock:
   - Required body fields: productId, variantSku, quantity, reason (optional)
   - Validate productId, variantSku, quantity are present; quantity must be positive integer
   - Find product and variant (same as deduct pattern)
   - Record stockBefore from current variant.stock
   - Atomic update: $inc variants.$.stock by +quantity, $inc variants.$.version by 1, with version match condition
   - On version conflict: return 409 with same error message pattern
   - On success: create InventoryAdjustment document with stockBefore, stockAfter, type 'restock', createdBy from req.user.userId
   - Return 200 with { success: true, stockBefore, stockAfter, version, adjustmentId }
   - Add Swagger JSDoc comment block matching the style of existing endpoints (see /deduct for reference)

4. Update GET /audit route to ALSO aggregate from InventoryAdjustment collection:
   - Add a third aggregation pipeline (alongside orderAudits and saleAudits):
     ```javascript
     const restockAudits = await InventoryAdjustment.aggregate([
       { $match: { productId: new mongoose.Types.ObjectId(productId), ...dateConditions } },
       { $project: {
           timestamp: '$createdAt',
           source: '$type',
           quantity: '$quantity',
           stockBefore: '$stockBefore',
           stockAfter: '$stockAfter',
           adjustmentId: '$_id',
           variantSku: '$variantSku'
         }
       }
     ]);
     ```
   - Note: InventoryAdjustment stores productId as a top-level field (not nested in items array), so NO $unwind is needed. The $match uses productId directly (unlike Order/Sale which use 'items.productId').
   - Add `const mongoose = require('mongoose');` at the top of the file if not already imported (needed for `new mongoose.Types.ObjectId()` in the aggregation)
   - Merge into the combined auditLog array: `[...orderAudits, ...saleAudits, ...restockAudits].sort(...)`
