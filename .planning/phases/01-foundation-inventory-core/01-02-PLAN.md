  <action>
Create Admin Mongoose schema with automatic password hashing:

**Schema fields:**
- username (String, required, unique, trim, lowercase)
- email (String, required, unique, lowercase)
- password (String, required, minlength: 8) - will store bcrypt hash, never plaintext
- role (String, enum: ['admin', 'staff'], default: 'admin')
- active (Boolean, default: true)
- lastLogin (Date)
- createdAt/updatedAt (timestamps: true)

**Pre-save hook for password hashing:**
- Before saving, check if password field is modified: `this.isModified('password')`
- If modified, hash password using bcrypt with salt rounds of 10: `bcrypt.hash(this.password, 10)`
- Replace plaintext password with bcrypt hash
- This ensures passwords are ALWAYS hashed before storage (satisfies AUTH-02)

**Instance method for password comparison:**
- Add `comparePassword(candidatePassword)` method
- Uses `bcrypt.compare()` to verify password against stored hash
- Returns Promise resolving to boolean (true if match, false otherwise)
- Used during login to validate credentials

**Indexes:**
- Index on username (unique)
- Index on email (unique)

**Export:**
`module.exports = mongoose.model('Admin', AdminSchema)`

**Important:** Do NOT store plaintext passwords. The pre-save hook must intercept and hash before database write.

<task type="auto">
  <name>Task 2: Create JWT authentication middleware</name>
  <files>
    api/middleware/auth.js
  </files>
  <action>
Create Express middleware for JWT token verification:

**authenticateToken function:**
- Extract token from Authorization header: `req.headers.authorization`
- Expected format: "Bearer {token}" - split and take second part
- If no token provided, return 401: `{ error: 'Access token required' }`
- Verify token using `jwt.verify(token, process.env.JWT_SECRET)`
- On successful verification, attach decoded payload to `req.user`
- On verification failure (expired, invalid, tampered), return 403: `{ error: 'Invalid or expired token' }`
- Call `next()` to continue to route handler if token valid

**Token payload structure:**
When generating tokens (in Plan 03's login endpoint), include:
- `userId`: Admin's ObjectId
- `username`: Admin's username
- `role`: Admin's role
- `iat`: Issued at timestamp (automatic)
- `exp`: Expiration timestamp (automatic)

**Error handling:**
- JsonWebTokenError: Token malformed → 403
- TokenExpiredError: Token expired → 403
- No token provided → 401

**Export:**
`module.exports = { authenticateToken }`

**Usage pattern:**
Routes will use this middleware like: `router.get('/protected', authenticateToken, handler)`
  </action>

Run `node -e "const { authenticateToken } = require('./api/middleware/auth.js'); console.log('Middleware loaded:', typeof authenticateToken === 'function')"` confirms middleware exports correctly.
  </verify>
  <done>
