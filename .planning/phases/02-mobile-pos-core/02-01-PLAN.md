<task type="auto" tdd="true">
  <name>Task 2: PIN Auth Endpoints</name>
  <files>api/routes/auth.js, api/tests/pin-auth.test.js</files>
  <behavior>
    - POST /api/auth/pin-setup: accepts { pin } (4-6 digit string), requires authenticateToken, hashes PIN with bcrypt (salt 10), stores on Admin.pinHash, returns 200 { message: 'PIN set successfully' }
    - POST /api/auth/pin-setup: returns 400 if pin missing or not 4-6 digits
    - POST /api/auth/pin-login: accepts { pin }, finds the single active admin with a pinHash set, compares with bcrypt, returns 200 { token, user } same shape as existing login
    - POST /api/auth/pin-login: returns 401 for wrong PIN with { error: 'Invalid PIN' }
    - POST /api/auth/pin-login: returns 400 if no admin has a PIN configured
    - JWT token from pin-login has 24h expiry (per user decision)
  </behavior>
  <action>
    Add two new routes to api/routes/auth.js:

    POST /pin-setup (protected by authenticateToken):
    1. Validate req.body.pin is a string of 4-6 digits (regex: /^\d{4,6}$/)
    2. Hash the PIN using bcrypt with salt rounds 10 (same pattern as password hashing)
    3. Update Admin document: admin.pinHash = hashedPin; await admin.save()
    4. Return 200 { message: 'PIN set successfully' }

    POST /pin-login (public):
    1. Validate req.body.pin exists
    2. Find single active admin with pinHash set: Admin.findOne({ active: true, pinHash: { $ne: null } })
    3. If no admin found with PIN: return 400 { error: 'No PIN configured. Use password login first to set up PIN.' }
    4. Compare using bcrypt.compare(pin, admin.pinHash)
    5. If match: update lastLogin, generate JWT with 24h expiry (same payload as password login), return { token, user }
    6. If no match: return 401 { error: 'Invalid PIN' }

    Use bcrypt for PIN hashing (not SHA-256) to stay consistent with the existing password hashing pattern in this codebase. The mobile app will send the raw PIN; the server hashes it.

    Write api/tests/pin-auth.test.js with TDD approach:
    - Test PIN setup with valid 4-digit PIN returns 200
    - Test PIN setup with valid 6-digit PIN returns 200
    - Test PIN setup rejects non-numeric PIN (400)
    - Test PIN setup rejects 3-digit PIN (400)
    - Test PIN setup requires auth token (401)
    - Test PIN login with correct PIN returns token + user
    - Test PIN login with wrong PIN returns 401
    - Test PIN login when no PIN configured returns 400
  </action>
  <verify>
    <automated>cd api && npx jest tests/pin-auth.test.js --no-cache</automated>
  </verify>
  <done>PIN setup endpoint stores bcrypt-hashed PIN on Admin. PIN login endpoint authenticates with PIN and returns 24h JWT. All 8+ tests pass.</done>
</task>
