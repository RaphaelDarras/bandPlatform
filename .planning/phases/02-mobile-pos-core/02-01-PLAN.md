  <behavior>
    PIN auth:
    - POST /api/auth/pin-setup with { pin: "1234" } (authenticated) stores bcrypt-hashed PIN on Admin, returns 200
    - POST /api/auth/pin-setup with pin shorter than 4 digits returns 400
    - POST /api/auth/pin-setup with pin longer than 6 digits returns 400
    - POST /api/auth/pin-setup without auth returns 401
    - POST /api/auth/pin-login with { pin: "1234" } returns { token, user } with 24h JWT
    - POST /api/auth/pin-login with wrong PIN returns 401
    - POST /api/auth/pin-login when no PIN set returns 400

    Concert CRUD:
    - GET /api/concerts returns array of concerts sorted by date desc
    - POST /api/concerts with { name, venue, date, city } creates concert (city maps to location field), returns 201
    - POST /api/concerts with missing required fields returns 400
    - GET /api/concerts/:id returns single concert
    - GET /api/concerts/:id with invalid id returns 404
    - PATCH /api/concerts/:id with { active: false } closes concert, returns updated concert
    - All concert endpoints require auth (401 without token)
  </behavior>
  <action>
    Add to api/routes/auth.js:
    - POST /pin-setup: Authenticated route. Validates pin is string of 4-6 digits only (regex /^\d{4,6}$/). Hash pin with bcrypt (salt 10) and store as admin.pinHash. Return 200 { message: 'PIN set successfully' }.
    - POST /pin-login: Public route. Accepts { pin }. Find the single admin account (Admin.findOne({ active: true })). Compare pin against admin.pinHash using bcrypt.compare. On match, generate 24h JWT with same payload as password login. Return { token, user }. On fail, return 401 { error: 'Invalid PIN' }. If no pinHash set, return 400 { error: 'PIN not configured' }.

    Create api/routes/concerts.js:
    - GET / : authenticateToken, Concert.find().sort({ date: -1 }). Return 200 with array.
    - POST / : authenticateToken, validate name + date + city required (venue optional). Create concert with city mapped to location field. Return 201.
    - GET /:id : authenticateToken, Concert.findById. Return 404 if not found.
    - PATCH /:id : authenticateToken, Concert.findByIdAndUpdate with $set for allowed fields (name, venue, date, location, active). Return 200 with updated doc.

    Register concerts route in api/index.js:
    - Add `app.use('/api/concerts', require('./routes/concerts'));`

    Create api/tests/pin-auth.test.js using existing test patterns (Jest + Supertest + mongoose connect/disconnect):
    - Test all PIN setup and login scenarios listed in behavior
