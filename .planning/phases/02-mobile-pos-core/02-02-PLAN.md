<task type="auto" tdd="true">
  <name>Task 1: Batch sale submission endpoint with idempotency</name>
  <files>api/routes/sales.js, api/index.js, api/tests/sales-batch.test.js</files>
  <behavior>
    Batch submission:
    - POST /api/sales/batch with array of 3 sales creates all 3, returns 201 with { created: 3, skipped: 0, sales: [...] }
    - POST /api/sales/batch with duplicate idempotencyKey skips duplicates, returns 201 with { created: 1, skipped: 2 }
    - POST /api/sales/batch deducts stock for each sale item via atomic $inc on Product.variants.$.stock
    - POST /api/sales/batch with empty array returns 400
    - POST /api/sales/batch without auth returns 401
    - Each sale in batch requires: concertId, items (array with productId, variantSku, quantity, priceAtSale), totalAmount, paymentMethod, idempotencyKey
    - stockBefore/stockAfter recorded per item using pre-read of variant stock

    Sale listing:
    - GET /api/sales returns all sales sorted by createdAt desc
    - GET /api/sales?concertId=xxx returns only sales for that concert
    - GET /api/sales includes voided status in response
  </behavior>
  <action>
    Create api/routes/sales.js:

    POST /batch:
    - authenticateToken middleware
    - Validate request body is non-empty array
    - For each sale in the batch:
      1. Check if idempotencyKey already exists in DB (Sale.findOne({ idempotencyKey })). If yes, skip (add to skipped count).
      2. For each item, read current stock: `Product.findOne({ 'variants.sku': item.variantSku })` to get stockBefore.
      3. Deduct stock atomically: `Product.findOneAndUpdate({ _id: item.productId, 'variants.sku': item.variantSku }, { $inc: { 'variants.$.stock': -item.quantity } }, { new: true })` to compute stockAfter.
      4. Build sale document with all fields including stockBefore/stockAfter per item, createdBy from req.user.userId.
      5. Save sale with Sale.create().
    - Concert sales never rejected (per user decision) — stock can go negative. Do NOT validate stock >= 0.
    - Return { created, skipped, sales } with 201.

    GET /:
    - authenticateToken middleware
    - If query.concertId, filter by concertId
    - Sort by createdAt desc
    - Return array of sales

    Register in api/index.js:
    - `app.use('/api/sales', require('./routes/sales'));`

    Write tests in api/tests/sales-batch.test.js following existing test patterns.
  </action>
  <verify>
    <automated>cd api && npm test -- --testPathPattern="sales-batch" --verbose</automated>
  </verify>
  <done>Batch endpoint creates sales, deduplicates by idempotencyKey, deducts stock atomically, listing works with concert filter</done>
</task>

<task type="auto" tdd="true">
  <name>Task 2: Void and unvoid sale endpoints</name>
  <files>api/routes/sales.js, api/tests/sales-batch.test.js</files>
  <behavior>
    Void:
    - POST /api/sales/:id/void sets voidedAt and voidedBy, reverses stock (adds back quantities), returns updated sale
    - POST /api/sales/:id/void on already voided sale returns 400 "Sale already voided"
    - POST /api/sales/:id/void with invalid id returns 404

    Unvoid:
    - POST /api/sales/:id/unvoid clears voidedAt and voidedBy, re-deducts stock, returns updated sale
    - POST /api/sales/:id/unvoid on non-voided sale returns 400 "Sale is not voided"
    - POST /api/sales/:id/unvoid with invalid id returns 404
  </behavior>
  <action>
    Add to api/routes/sales.js:

    POST /:id/void:
    - authenticateToken
    - Find sale by id. Return 404 if not found.
    - If sale.voidedAt already set, return 400 { error: 'Sale already voided' }.
    - For each item in sale.items, reverse stock: `Product.findOneAndUpdate({ _id: item.productId, 'variants.sku': item.variantSku }, { $inc: { 'variants.$.stock': item.quantity } })`.
    - Set sale.voidedAt = new Date(), sale.voidedBy = req.user.userId. Save.
    - Return 200 with updated sale.

    POST /:id/unvoid:
    - authenticateToken
    - Find sale by id. Return 404 if not found.
    - If !sale.voidedAt, return 400 { error: 'Sale is not voided' }.
    - For each item in sale.items, re-deduct stock: `Product.findOneAndUpdate({ _id: item.productId, 'variants.sku': item.variantSku }, { $inc: { 'variants.$.stock': -item.quantity } })`.
    - Unset voidedAt and voidedBy (set to undefined). Save.
    - Return 200 with updated sale.

    Add void/unvoid tests to api/tests/sales-batch.test.js.
  </action>
  <verify>
    <automated>cd api && npm test -- --testPathPattern="sales-batch" --verbose</automated>
  </verify>
  <done>Void sets voidedAt + reverses stock, unvoid clears void + re-deducts stock, edge cases handled with proper status codes</done>
</task>
