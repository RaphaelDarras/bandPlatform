  <action>
    Create mobile/src/features/cart/useSaleRecording.ts:
    - Hook that encapsulates the sale recording flow
    - recordSale():
      1. Read cart items, concertId, currency, discount, discountType from cartStore
      2. Generate UUID for sale ID and idempotencyKey (type=sale_create + saleId)
      3. Build LocalSale object matching SQLite sales table schema
      4. Build OutboxEntry with type='sale_create', payload=JSON.stringify(sale data for API batch format)
      5. Call recordSaleLocally(db, sale, outboxEntry) — atomic SQLite write
      6. Update local product stock: for each item, call updateLocalStock(db, productId, variantSku, -quantity)
      7. Call cartStore.clearCart()
      8. Update syncStore.pendingCount (increment by 1)
      9. Return sale.id
    - All operations are local — no API calls. The outbox drives sync later.

    Create mobile/src/app/selling/review.tsx:
    - Read items, total, currency, discount, discountType from cartStore
    - Item list (read-only, can tap back to modify in cart)
    - Payment method: 4 buttons (Cash, Card, E-transfer, PayPal) — radio-style selection, stores in local state
    - Currency: dropdown or horizontal chip selector — calls cartStore.setCurrency on change
    - Discount: TextInput for amount + toggle for flat/percent — calls cartStore.setDiscount on change
    - Grand total: computed from cartStore.total() (already includes discount)
    - "Confirm Sale" button:
      1. Set loading state
      2. Call useSaleRecording().recordSale()
      3. Show success toast ("Sale recorded!")
      4. Navigate back: router.replace('/selling') to return to product grid
    - Per-concert price overrides: If concert has overrides in SQLite concert_price_overrides table, those prices are already reflected in the cart items' priceAtSale (set at tile-tap time). Review screen displays the overridden price.
