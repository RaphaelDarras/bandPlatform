  <files>mobile/src/features/auth/pinAuth.ts, mobile/src/features/auth/useAuth.ts, mobile/src/app/(auth)/pin.tsx, mobile/src/app/(auth)/_layout.tsx, mobile/src/app/_layout.tsx</files>
  <action>
    Create mobile/src/features/auth/pinAuth.ts:
    1. Import expo-secure-store and expo-crypto
    2. Constants: PIN_HASH_KEY = 'pin_hash', JWT_KEY = 'auth_token', JWT_EXPIRY_KEY = 'auth_token_expiry'
    3. hashPin(pin: string): Promise<string> — use Crypto.digestStringAsync(SHA256, pin)
    4. setupPinLocally(pin: string, jwt: string): stores SHA-256 hash of PIN and JWT in SecureStore. Also store JWT expiry (Date.now() + 24h in ms) in SecureStore.
    5. verifyPinOffline(inputPin: string): Promise<boolean> — compare hash of input against stored hash
    6. getCachedToken(): Promise<string | null> — return stored JWT if not expired (check expiry), null otherwise
    7. hasPinConfigured(): Promise<boolean> — check if PIN_HASH_KEY exists in SecureStore
    8. clearCredentials(): clear all SecureStore keys

    Create mobile/src/features/auth/useAuth.ts:
    1. Custom hook useAuth() that provides:
       - login(pin: string): async function
         a. If online: POST /api/auth/pin-login with { pin }. On success: call authStore.setAuth(token, user.id), call setupPinLocally(pin, token). On 401: throw 'Invalid PIN'.
         b. If offline: call verifyPinOffline(pin). If match: get cached token with getCachedToken(). If token valid: call authStore.setAuth(cachedToken, cached userId). If no cached token: still allow login but set a flag for "offline session" (degraded mode — no sync until online).
       - logout(): clear authStore, clear SecureStore credentials
       - isLoading: boolean state during auth check
       - error: string | null for displaying error messages
    2. Use useSyncStore for isOnline check
    3. Cold-start offline handling (Claude's discretion): If hasPinConfigured() returns false and device is offline, show message "Connect to internet to set up your PIN for the first time"

    Create mobile/src/app/(auth)/_layout.tsx:
    - Simple Stack layout for auth screens

    Create mobile/src/app/(auth)/pin.tsx:
    - PIN entry screen with:
      - App logo/name at top
      - "Enter your PIN" text (i18n key: auth.enterPin)
      - 4-6 digit numeric input — use a row of filled/empty circles (like phone unlock) rather than a visible text input
      - Custom numeric keypad (0-9 digits + backspace + confirm/enter button)
      - One-handed operation friendly (bottom-aligned keypad)
      - Error message display area (shows "Wrong PIN, try again" on failure — no lockout per user decision)
      - Loading state during online verification
      - Offline indicator if device is offline
      - If no PIN configured and offline: show setup-required message with retry button
    - NativeWind styling: dark-friendly, large touch targets (min 48x48px), high contrast
    - On successful login: router.replace('/(tabs)/')
