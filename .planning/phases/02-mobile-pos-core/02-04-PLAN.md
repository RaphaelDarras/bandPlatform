  <name>Task 3: Session expiry warning (alert 30 minutes before JWT expires)</name>
  <files>mobile/src/features/auth/sessionExpiry.ts, mobile/src/__tests__/features/auth/sessionExpiry.test.ts, mobile/src/features/auth/useAuth.ts</files>
  <behavior>
    sessionExpiry.ts:
    - parseJwtExp(token) extracts exp claim from JWT payload (base64 decode, no verification needed — server already verified)
    - parseJwtExp returns Unix timestamp (seconds) or null if malformed
    - parseJwtExp("eyJhbGciOiJIUzI1NiJ9.eyJleHAiOjE3MTAwMDAwMDB9.xxx") returns 1710000000
    - parseJwtExp("not-a-jwt") returns null
    - scheduleExpiryWarning(token, onWarning) calculates (exp - 30min) and sets a setTimeout
    - scheduleExpiryWarning returns a cleanup function that clears the timeout
    - If token expires in less than 30 minutes, fires warning immediately
    - If token is already expired, fires warning immediately
    - clearExpiryWarning() cancels any scheduled timer

    Integration with useAuth:
    - On successful login (online or offline with cached token): call scheduleExpiryWarning with the JWT
    - Warning callback: show Alert.alert prompting user to re-authenticate ("Session expiring soon. Please log in again to continue.")
    - On logout: call clearExpiryWarning
  </behavior>
  <action>
    Create mobile/src/features/auth/sessionExpiry.ts:
    - parseJwtExp(token: string): number | null
      - Split token by '.', take payload (index 1)
      - Base64-decode payload (use atob or Buffer depending on RN environment — prefer a manual base64 decode to avoid polyfill issues: use a simple base64url decode function)
      - Parse JSON, extract 'exp' field
      - Return exp as number or null if any step fails
    - scheduleExpiryWarning(token: string, onWarning: () => void): () => void
      - Parse exp from token
      - If null, return no-op cleanup
      - Calculate warningTime = (exp * 1000) - (30 * 60 * 1000) — 30 minutes before expiry in ms
      - Calculate delay = warningTime - Date.now()
      - If delay <= 0, call onWarning() immediately, return no-op
      - Set timeout with delay, call onWarning when fired
      - Return cleanup function that clears the timeout
    - clearExpiryWarning(): void — module-level timer reference, clears it

    Update mobile/src/features/auth/useAuth.ts:
    - Import scheduleExpiryWarning, clearExpiryWarning from sessionExpiry
    - After successful login (both online and offline paths), call:
      ```
      clearExpiryWarning(); // clear any previous timer
      scheduleExpiryWarning(token, () => {
        Alert.alert(
          t('auth.sessionExpiring'),
          t('auth.sessionExpiringMessage'),
          [{ text: t('common.ok') }]
        );
      });
      ```
    - In logout function, add clearExpiryWarning() call
    - Add i18n keys: auth.sessionExpiring = "Session Expiring", auth.sessionExpiringMessage = "Your session expires in 30 minutes. Please log in again to stay connected."
