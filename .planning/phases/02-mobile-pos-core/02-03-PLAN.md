<task type="auto" tdd="true">
  <name>Task 2: SQLite data layer with outbox pattern and Zustand stores</name>
  <files>mobile/src/db/index.ts, mobile/src/db/migrations.ts, mobile/src/db/outbox.ts, mobile/src/db/sales.ts, mobile/src/stores/cartStore.ts, mobile/src/stores/authStore.ts, mobile/src/stores/syncStore.ts, mobile/src/i18n/index.ts</files>
  <behavior>
    Database:
    - getDb() returns SQLite database instance
    - runMigrations creates tables: sales, outbox, products, concerts, concert_price_overrides
    - recordSaleLocally writes sale + outbox entry atomically in single transaction
    - getPendingOutboxRows returns rows with status='pending' and next_attempt_at <= now
    - markOutboxDone updates row status to 'done'

    Cart store:
    - addItem adds CartItem to items array
    - removeItem removes by variantSku
    - updateQuantity changes quantity for specific item
    - clearCart resets items and discount
    - total() with no discount returns sum of priceAtSale * quantity
    - total() with flat discount of 5 on subtotal 20 returns 15
    - total() with percent discount of 10 on subtotal 100 returns 90
    - total() with flat discount larger than subtotal returns 0 (not negative)
    - setConcertId sets the active concert
    - setCurrency sets per-sale currency (default EUR)

    Auth store:
    - setToken stores JWT and sets isAuthenticated true
    - clearAuth clears token and sets isAuthenticated false
    - getToken returns stored token

    Sync store:
    - setIsOnline updates online status
    - setPendingCount updates count
    - setLastSyncAt updates timestamp
  </behavior>
  <action>
    Create mobile/src/db/index.ts:
    - Singleton getDb() function using expo-sqlite openDatabaseAsync('pos.db')
    - Enable WAL mode: `db.execAsync('PRAGMA journal_mode = WAL;')`
    - Call runMigrations on first open
    - Follow Pattern from research (SQLite Database Initialization)

    Create mobile/src/db/migrations.ts:
    - CREATE TABLE IF NOT EXISTS for: sales, outbox, products, concerts, concert_price_overrides
    - Use exact schemas from research (Local Sales Table Schema, Outbox table)
    - Add products table: id TEXT PK, name TEXT, price REAL, variants_json TEXT, image_url TEXT, active INTEGER DEFAULT 1, updated_at INTEGER
    - Add concerts table: id TEXT PK, name TEXT, date INTEGER, venue TEXT, city TEXT, active INTEGER DEFAULT 1, updated_at INTEGER

    Create mobile/src/db/outbox.ts:
    - recordSaleLocally: atomic db.withTransactionAsync writing to both sales and outbox tables
    - getPendingOutboxRows: SELECT from outbox WHERE status='pending' AND next_attempt_at <= ?
    - markOutboxDone: UPDATE outbox SET status='done' WHERE id=?
    - incrementAttempt: UPDATE outbox SET attempt_count=attempt_count+1, next_attempt_at=? WHERE id=?
    - Follow Pattern 1 from research exactly

    Create mobile/src/db/sales.ts:
    - getLocalSales(concertId?: string): query sales table with optional concert filter, sorted by created_at DESC
    - voidLocalSale(id: string): set voided=1, voided_at=Date.now()
    - unvoidLocalSale(id: string): set voided=0, voided_at=null

    Create mobile/src/stores/cartStore.ts:
    - Follow Pattern 5 from research (Zustand + MMKV)
    - CartItem: { productId, variantSku, productName, variantLabel, quantity, priceAtSale }
    - Include concertId, currency (default 'EUR'), discount, discountType ('flat'|'percent')
    - total() computed as getter — applies discount after summing items
    - Flat discount: Math.max(0, subtotal - discount)
    - Percent discount: subtotal * (1 - discount/100)

    Create mobile/src/stores/authStore.ts:
    - Zustand store with MMKV persistence
    - Fields: token, isAuthenticated, user (id, username, role)
    - Actions: setToken(token, user), clearAuth()

    Create mobile/src/stores/syncStore.ts:
    - Zustand store with MMKV persistence
    - Fields: isOnline, pendingCount, lastSyncAt, consecutiveFailures
    - Actions: setIsOnline, setPendingCount, setLastSyncAt, incrementFailures, resetFailures

    Create mobile/src/i18n/index.ts + en.json + fr.json:
    - Follow i18n setup from research
    - Minimal initial translations: app name, dashboard, history, stock, settings, common buttons (confirm, cancel, back)

    Write unit tests:
    - mobile/src/__tests__/db/outbox.test.ts: Mock expo-sqlite, test recordSaleLocally calls withTransactionAsync with correct INSERT statements, test getPendingOutboxRows query, test markOutboxDone
    - mobile/src/__tests__/stores/cartStore.test.ts: Test addItem, removeItem, updateQuantity, clearCart, total with no discount, flat discount, percent discount, discount exceeding subtotal
  </action>
