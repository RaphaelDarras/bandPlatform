<task type="auto" tdd="true">
  <name>Task 2: SQLite Database + Migrations + Outbox</name>
  <files>mobile/src/db/index.ts, mobile/src/db/migrations.ts, mobile/src/db/outbox.ts, mobile/src/db/sales.ts, mobile/src/db/products.ts, mobile/src/db/concerts.ts</files>
  <behavior>
    - getDb() returns SQLite database singleton with WAL mode enabled
    - migrations create tables: sales, outbox, products_cache, concerts_cache, concert_price_overrides
    - recordSaleLocally() writes to sales + outbox in single atomic transaction
    - outbox has idempotency_key unique index
    - getPendingOutbox() returns pending rows ordered by created_at ASC with LIMIT
    - markOutboxDone(id) sets status to 'done'
    - getLocalSales(concertId?) queries sales table
    - products_cache table stores product catalog for offline use
    - concerts_cache table stores concert list for offline use
  </behavior>
  <action>
    Create src/db/index.ts:
    - Singleton pattern using module-level variable
    - openDatabaseAsync('pos.db')
    - Enable WAL: PRAGMA journal_mode = WAL
    - Call runMigrations on first open
    - Export getDb() async function

    Create src/db/migrations.ts:
    - Export async runMigrations(db) function
    - Create sales table: id TEXT PK, concert_id TEXT NOT NULL, items_json TEXT NOT NULL, total_amount REAL NOT NULL, payment_method TEXT NOT NULL, currency TEXT DEFAULT 'EUR', discount REAL DEFAULT 0, discount_type TEXT DEFAULT 'flat', voided INTEGER DEFAULT 0, voided_at INTEGER, synced INTEGER DEFAULT 0, created_at INTEGER NOT NULL
    - Create outbox table: id TEXT PK, type TEXT NOT NULL, payload TEXT NOT NULL, idempotency_key TEXT NOT NULL, status TEXT DEFAULT 'pending', attempt_count INTEGER DEFAULT 0, next_attempt_at INTEGER DEFAULT 0, created_at INTEGER NOT NULL. Unique index on idempotency_key.
    - Create products_cache table: id TEXT PK, data_json TEXT NOT NULL, updated_at INTEGER NOT NULL
    - Create concerts_cache table: id TEXT PK, data_json TEXT NOT NULL, updated_at INTEGER NOT NULL
    - Create concert_price_overrides table: concert_id TEXT, product_id TEXT, price REAL NOT NULL, PRIMARY KEY (concert_id, product_id)
    - Create indexes: sales_concert_idx on sales(concert_id), sales_created_idx on sales(created_at DESC), outbox_status_idx on outbox(status, next_attempt_at)

    Create src/db/outbox.ts:
    - Export recordSaleLocally(db, sale, outboxEntry) — uses db.withTransactionAsync to atomically insert into both sales and outbox tables (INSERT OR IGNORE for outbox to handle retries)
    - Export getPendingOutbox(db, limit=10) — SELECT from outbox WHERE status='pending' AND next_attempt_at <= now ORDER BY created_at ASC LIMIT ?
    - Export markOutboxDone(db, id) — UPDATE outbox SET status='done' WHERE id=?
    - Export updateOutboxRetry(db, id, attemptCount) — UPDATE with incremented attempt_count and exponential backoff next_attempt_at

    Create src/db/sales.ts:
    - Export getLocalSales(db, concertId?) — query sales table, optionally filtered by concert_id, ordered by created_at DESC
    - Export voidLocalSale(db, saleId) — UPDATE sales SET voided=1, voided_at=Date.now()
    - Export unvoidLocalSale(db, saleId) — UPDATE sales SET voided=0, voided_at=null

    Create src/db/products.ts:
    - Export cacheProducts(db, products[]) — REPLACE INTO products_cache for each product (upsert)
    - Export getCachedProducts(db) — SELECT all, parse data_json

    Create src/db/concerts.ts:
    - Export cacheConcerts(db, concerts[]) — REPLACE INTO concerts_cache
    - Export getCachedConcerts(db) — SELECT all, parse data_json
    - Export setPriceOverride(db, concertId, productId, price) — INSERT OR REPLACE into concert_price_overrides
    - Export getPriceOverrides(db, concertId) — SELECT all overrides for a concert
    - Export clearPriceOverrides(db, concertId) — DELETE FROM concert_price_overrides WHERE concert_id=?

    NOTE: Do NOT use any mocking libraries for expo-sqlite in tests. Instead, write pure unit tests for the SQL generation logic or integration tests that verify the function signatures and types. The actual SQLite calls will be tested via manual verification on device.

  <name>Task 3: Zustand Stores + API Client</name>
  <files>mobile/src/stores/authStore.ts, mobile/src/stores/cartStore.ts, mobile/src/stores/syncStore.ts, mobile/src/api/client.ts</files>
  <behavior>
    - authStore: isAuthenticated, token, setToken, clearAuth. Persisted with MMKV.
    - cartStore: items[], concertId, currency, discount, discountType. addItem merges quantity if same SKU already in cart. removeItem by SKU. updateQuantity. clearCart. total() computed with flat/percent discount. Persisted with MMKV.
    - syncStore: isOnline, pendingCount, lastSyncAt, consecutiveFailures, syncError. NOT persisted (runtime only).
    - API client: axios instance with baseURL from env/constants, request interceptor that reads JWT from authStore and attaches as Bearer token
    - Cart total with flat discount: max(0, subtotal - discount)
    - Cart total with percent discount: subtotal * (1 - discount/100)
  </behavior>
  <action>
    Create src/stores/authStore.ts:
    - Zustand store with MMKV persistence (id: 'auth-store')
    - State: isAuthenticated (boolean), token (string | null), userId (string | null)
    - Actions: setAuth(token, userId), clearAuth()
    - On setAuth: also store token in expo-secure-store for extra security (MMKV for fast hydration, SecureStore as source of truth)

    Create src/stores/cartStore.ts:
    - Follow the pattern from RESEARCH.md Pattern 5 exactly
    - Zustand store with MMKV persistence (id: 'cart-store')
    - CartItem interface: { productId: string, variantSku: string, name: string, variantLabel: string, quantity: number, priceAtSale: number }
    - State: items: CartItem[], concertId: string | null, currency: string (default 'EUR'), discount: number (default 0), discountType: 'flat' | 'percent' (default 'flat')
    - Actions:
      - addItem(item): if item with same variantSku exists, increment quantity; otherwise append
      - removeItem(variantSku): filter out
      - updateQuantity(variantSku, quantity): update quantity, remove if 0
      - setCurrency(code): update currency
      - setDiscount(amount, type): update discount and discountType
      - setConcertId(id): set active concert
      - clearCart(): reset items, discount to defaults (keep concertId and currency)
    - Computed: total() — subtotal of (priceAtSale * quantity) for each item, minus flat discount or times (1 - percent/100)
    - itemCount() — sum of all quantities

    Create src/stores/syncStore.ts:
    - Zustand store WITHOUT persistence (runtime state only)
    - State: isOnline: boolean (default false), pendingCount: number (default 0), lastSyncAt: number | null, consecutiveFailures: number (default 0), syncError: string | null
    - Actions: setIsOnline(val), setPendingCount(n), recordSyncSuccess(), recordSyncFailure(error), clearSyncError()

    Create src/api/client.ts:
    - Create axios instance with baseURL from expo-constants or environment config
    - Default baseURL: 'http://localhost:5000/api' (configurable via app.json extra or .env)
    - Request interceptor: read token from authStore (Zustand getState(), not hook). If token exists, set Authorization: Bearer {token}
    - Response interceptor: on 401/403, call authStore.getState().clearAuth() to force re-login
    - Export the axios instance as default
    - Export typed wrapper functions: api.get<T>(url), api.post<T>(url, data) for convenience

    Write unit tests for cartStore total calculation:
    - Test total with no items returns 0
    - Test total with items no discount
    - Test total with flat discount
    - Test total with percent discount
    - Test addItem merges quantity for same SKU
    - Test clearCart resets items but keeps concertId
  </action>
